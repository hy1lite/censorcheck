# Network Censorship Checker

![Preview](img/img3.jpg)

Небольшой скрипт-чекер для диагностики блокировок и цензуры на сервере.  
Особенно полезен для VPS/VDS, с локацией РФ.

Скрипт анализирует, заблокирован ли тестируемый сервис на различных уровнях:

- **DNS**
- **IP**
- **HTTP/HTTPS**
- **DPI**
- **REGIONAL**

Результаты выводятся в удобной таблице.

## 📌 Возможности

- Проверка доменов на тип блокировки.
- Встроенные списки сайтов, которые часто блокируются DPI или имеют географические ограничения.  
  (Список будет обновляться.)
- Автоматическая установка недостающих зависимостей (`curl`, `nslookup`, `nc`, `netcat`, `python3`).
- Определение статуса для каждого домена:
  - **OK** — доступен  
  - **BLOCKED** — заблокирован (с указанием типа блокировки)  
  - **PARTIAL** — частично доступен  
- **Радар ТСПУ:** Проверка доступности вашего сервера "снаружи" — из реальных домашних и мобильных сетей РФ (через сеть зондов RIPE Atlas).  
  Позволяет выявить ситуации, когда сервер доступен не у всех провайдеров.
- **Эмуляция Reality:** Зонды выполняют TLS-подключение к вашему серверу (`TLS Client Hello`) с заданным SNI, что позволяет проверить возможные проблемы на этапе TLS handshake (в том числе связанные с DPI).
- **Асинхронность:** Оптимизированная очередь запросов, исключающая зависания скрипта.

## ⚠️ Важно

1. Точность определения блокировок сайтов составляет **±90%**. Рекомендуется ручная перепроверка.
2. **Для работы Радара ТСПУ:** На вашем сервере **обязательно** должен быть открыт порт `443` (по IPv4), и на нем должна работать программа, умеющая отвечать на TLS-запросы. Это может быть установленный VPN (**Xray, 3X-UI, Marzban**) или обычный веб-сервер (**Nginx, Apache**). На абсолютно "чистом" сервере с закрытыми портами радар работать не будет.
